# M3 エディタ・画像 UX 仕様

> 対象マイルストーン: M3（エディタ＋画像）
> 作成日: 2026-06-14
> 担当: designer

---

## 1. 概要 / 目的

`/pages/[id]` にある本文プレースホルダを BlockNote ブロック型エディタに置き換え、
ユーザーが Notion 風のリッチなドキュメントを作成・編集できるようにする。
M2 で実装済みの PageHeader（アイコン＋タイトル）との一体感を保ちながら、
debounce 自動保存・画像挿入・`/` コマンドメニュー・フローティングツールバーを提供する。

---

## 2. 画面構成

### 2-1. デスクトップ（md 以上）

```
┌────────────────────────────────────────────────────────────────────────┐
│ [AppShell トップバー: h-10]  ≡ Menu  [auto-save status: 保存済み ✓]    │
├──────────────┬─────────────────────────────────────────────────────────┤
│              │         [エディタ領域: flex-1 overflow-auto]             │
│  サイドバー   │  ┌──────────── max-w-[900px] mx-auto ─────────────┐    │
│  260px       │  │  px-8 md:px-[72px]  pt-12  pb-24               │    │
│  （M2 実装済）│  │                                                  │    │
│              │  │  [アイコン] 📄               （PageHeader: M2済）│    │
│              │  │  [タイトル入力欄]                                 │    │
│              │  │  ─────────────────────────────── (区切り: 薄い) │    │
│              │  │                                                  │    │
│              │  │  [BlockNote エディタ本文]                        │    │
│              │  │   ↑ ここが M3 で実装される領域                   │    │
│              │  │                                                  │    │
│              │  └──────────────────────────────────────────────────┘   │
└──────────────┴─────────────────────────────────────────────────────────┘
```

- **最大幅**: `max-w-[900px]`、`mx-auto` で中央揃え
- **左右余白**: `px-8`（モバイル）/ `md:px-[72px]`（デスクトップ）
  - Notion は実測で左右 96px 程度。900px 幅コンテンツ域を確保しながら余白を持たせる
- **上余白**: PageHeader の `pt-12` は M2 で実装済み。エディタ本文は PageHeader の直下から始まる
- **下余白**: `pb-24`（モバイルキーボード出現時にコンテンツが隠れないよう余裕を持たせる）
- **自動保存ステータス**: AppShell トップバーの右端（詳細は §5）

### 2-2. モバイル（md 未満）

```
┌──────────────────────────────────────────┐
│ [トップバー: h-10]  ≡  [保存済み ✓]      │
├──────────────────────────────────────────┤
│   [アイコン] 📄                           │
│   [タイトル入力欄]                         │
│   ─────────────────────────────          │
│                                          │
│   [BlockNote エディタ本文]                │
│   （px-4 pt-8 pb-24）                    │
│                                          │
└──────────────────────────────────────────┘
```

- 1カラム。サイドバーはドロワー（M2 実装済み）
- エディタの左右余白はモバイルでは `px-4`（デスクトップより狭く、読みやすい幅を確保）
- フローティングツールバー・スラッシュメニューはモバイルでも同様に表示（ポジション: 画面内に収まるよう調整）

### 2-3. PageHeader との関係

PageHeader（`components/page/PageHeader.tsx`）は M2 実装済みで変更しない。
エディタはその直下に配置し、視覚的な一体感を出すために次のルールを守る:

- PageHeader と エディタの間に明示的な区切り線は引かない（余白のみで分離）
- ただしエディタ本文開始位置 `mt-2`（≒ タイトルと本文の余白）を設ける

## 3. コンポーネント分割

### 3-1. 新規作成するコンポーネント

| コンポーネント | 配置 | SC/CC | 役割 |
|--------------|------|-------|------|
| `components/editor/Editor.tsx` | ページ最下層のリーフ | **Client** (`'use client'`) | BlockNote エディタ本体。`next/dynamic`（`ssr:false`）でラップして読み込む。`content` props を受け取り、変更時に `onContentChange` コールバックを呼ぶ |
| `components/editor/EditorSkeleton.tsx` | — | Server 可 | BlockNote 読み込み中のスケルトン UI（`next/dynamic` の `loading:` に指定） |
| `components/editor/AutoSaveStatus.tsx` | AppShell トップバー内 | **Client** | `'saving' / 'saved' / 'error'` の状態を受け props で表示するプレゼンテーションコンポーネント |
| `components/editor/ImageUploadPlaceholder.tsx` | エディタ内 | **Client** | 画像アップロード中・失敗時のブロック内プレースホルダ |
| `hooks/use-autosave.ts` | — | **Client** Hook | debounce + Server Action 呼び出し + 状態管理 |

### 3-2. 変更するコンポーネント

| コンポーネント | 変更内容 |
|--------------|---------|
| `components/page/PageView.tsx` | プレースホルダ段落を `Editor` コンポーネント + `AutoSaveStatus` に差し替える |
| `app/(main)/layout.tsx` | AppShell に `AutoSaveStatus` 表示スロットを追加（詳細は §5） |

### 3-3. props の概要

#### `Editor` props

```
pageId: string          // Server Action 呼び出し用のページID
initialContent: unknown // DB から取得した BlockNote JSON（型は BlockNote の Block[]）
onSaveStatusChange: (status: 'saving' | 'saved' | 'error') => void
                        // 保存状態の変化を親（PageView）へ通知する
```

`Editor` 内部で `useAutosave` フックを使い、コンテンツ変更のたびに debounce 保存を行う。

#### `AutoSaveStatus` props

```
status: 'saving' | 'saved' | 'error' | 'idle'
onRetry: () => void   // エラー時の「再試行」クリックハンドラ
```

#### `EditorSkeleton` props

なし。固定の高さ（例: `h-48`）でローディング表示するだけ。

## 4. データの流れ

```
Server Component (app/(main)/pages/[id]/page.tsx)
  └─ getPage(id) → PageDetail（content: BlockNote JSON）
      └─ PageView（Client Component）
          ├─ PageHeader（icon, title: M2 実装済み）
          └─ Editor（initialContent: PageDetail.content）
              └─ useAutosave
                  └─ updatePageContent({ id, content, contentText })  ← Server Action（M2 実装済み）
```

- `initialContent` は Server Component 側で取得済みの BlockNote JSON をそのまま渡す（Client から再フェッチしない）
- `content_text`（プレーンテキスト）は BlockNote の API（`editor.document` を走査）でブラウザ側で抽出し、Server Action に同送する

## 5. 自動保存ステータス表示

### 5-1. 配置

AppShell トップバー（`header` 要素、`h-10`）の**右端**。
現在のトップバーは `≡` メニューボタンのみで右端が空いているため、ここに配置する。

```
┌──────────────────────────────────────────────────────────────────┐
│  ≡  [（モバイルはドロワートリガー）]          [保存済み ✓]       │
└──────────────────────────────────────────────────────────────────┘
  ↑ AppShell の header 要素。左端はボタン、右端がステータス
```

ステータスはエディタページ（`/pages/[id]`）でのみ表示する。
他のページ（設定・ごみ箱等）では非表示。

### 5-2. 状態と表示

| 状態 | アイコン | 文言 | 色 | 補足 |
|------|---------|------|-----|------|
| `idle` | なし | （非表示） | — | 初期状態、まだ変更なし |
| `saving` | `Loader2`（回転） | 保存中… | `text-muted-foreground` | debounce 発火後、Server Action 完了前 |
| `saved` | `Check` | 保存済み | `text-muted-foreground` | Server Action 成功後。3秒後に `idle` へ遷移 |
| `error` | `AlertCircle` | 保存に失敗しました | `text-destructive` | Server Action 失敗後。再試行ボタン併記 |

### 5-3. 失敗時の再試行導線

`error` 状態のとき、ステータス表示の隣に「**再試行**」テキストボタン（`variant="link"` 相当・`text-sm`）を表示。
クリックすると即時に保存を再実行する（debounce をバイパスして呼ぶ）。
再試行中は `saving` へ、成功したら `saved` へ遷移する。

自動リトライはしない。ユーザーが明示的に再試行するか、次の入力で再度 debounce が走ったときに保存される。

### 5-4. トランジション

- `idle` → `saving`: 即時
- `saving` → `saved`: Server Action 完了と同時。フェードインアニメーション（`transition-opacity duration-200`）
- `saved` → `idle`: 3秒後に自動フェードアウト
- `saving` → `error`: Server Action 失敗と同時
- `error` → `saving`: 再試行クリック時

### 5-5. アクセシビリティ

- ステータス変化は `role="status"` + `aria-live="polite"` で読み上げる
- エラー時は `role="alert"` + `aria-live="assertive"` に切り替える
- 「再試行」ボタンには `aria-label="自動保存を再試行"` を付与

## 6. 対応ブロック

### 6-1. ブロック一覧と実装方針

| ブロック | BlockNote 標準 | カスタム実装 | 説明 |
|---------|--------------|------------|------|
| 段落 | ✓ | — | デフォルトブロック |
| 見出し H1 | ✓ | — | BlockNote の `heading` level 1 |
| 見出し H2 | ✓ | — | level 2 |
| 見出し H3 | ✓ | — | level 3 |
| 箇条書きリスト | ✓ | — | `bulletListItem`。Tab でネスト、Shift+Tab でアウトデント |
| 番号付きリスト | ✓ | — | `numberedListItem`。同上 |
| チェックリスト | ✓ | — | `checkListItem`。チェック状態を content に保存 |
| 引用 | — | **要カスタム** | BlockNote 標準に `quote` ブロックなし |
| コードブロック | ✓（要確認） | 要拡張 | 言語指定＋シンタックスハイライト（`CodeBlock` プラグイン。言語セレクタ付き） |
| 区切り線 | — | **要カスタム** | `---` 相当の水平線ブロック |
| 画像 | ✓（要拡張） | 要拡張 | デフォルトの image ブロックをカスタムして Supabase Storage 連携にする |

**backend に要相談**: カスタムブロック（引用・区切り線）は BlockNote の `createReactBlockSpec` で実装する。JSONスキーマの形式・`content` カラムへの保存形式は frontend エンジニアが決定してよいが、後続の検索機能（M5 以降）で `content_text` 抽出が正しくできるか確認すること。

### 6-2. 各ブロックの見た目仕様

#### 段落

- フォントサイズ: `text-base`（16px）
- 行間: `leading-relaxed`（1.625）
- 色: `text-foreground`
- 上下余白: ブロック間 `my-1`（BlockNote デフォルトに委ねる）

#### 見出し H1 / H2 / H3

| レベル | サイズ | ウェイト | 上余白 | 下余白 |
|--------|--------|---------|--------|--------|
| H1 | `text-3xl` | `font-bold` | `mt-8` | `mb-2` |
| H2 | `text-2xl` | `font-semibold` | `mt-6` | `mb-2` |
| H3 | `text-xl` | `font-semibold` | `mt-4` | `mb-1` |

色: `text-foreground`。ページタイトルの `text-3xl font-bold`（PageHeader）と被るため、
H1 はページタイトルより一回り小さく見えるよう、タイトルが `text-3xl`（30px）に対し H1 は `text-2xl`（24px）とする調整も検討すること（frontend 判断）。

#### 箇条書き / 番号付きリスト

- 先頭マーカー: `text-muted-foreground`
- ネストインデント: 1段あたり `ml-6`
- ネストマーカー: 1段 `●`、2段 `○`、3段 `▪`（BlockNote デフォルトに委ねる）

#### チェックリスト

- 未チェック: `☐`（四角形アイコン or shadcn/ui の `Checkbox`）
- チェック済み: `☑`。テキストに `line-through text-muted-foreground/60`

#### 引用（カスタムブロック）

```
┌──────────────────────────────────────────┐
│ ▌  引用テキストはここに入力する           │
│    複数行も可能                           │
└──────────────────────────────────────────┘
```

- 左端に `border-l-4 border-muted-foreground/40`
- 背景: なし（Notion 風のシンプルさ）
- テキスト色: `text-muted-foreground`
- パディング: `pl-4 py-1`
- `createReactBlockSpec` で実装（**要カスタム**と前節に明記）

#### コードブロック

```
┌──────────────────────────────────────────┐
│ [言語セレクタ: typescript ▼]             │
│─────────────────────────────────────────│
│ const x = 1;                             │
│ console.log(x);                          │
└──────────────────────────────────────────┘
```

- 背景: `bg-muted`（ライト）/ `bg-muted`（ダーク）
- フォント: `font-mono text-sm`
- 丸み: `rounded-md`
- 言語セレクタ: 上部右寄せ or 左寄せのドロップダウン（`text-xs text-muted-foreground`）
- シンタックスハイライト: BlockNote の `codeBlock` プラグイン + lowlight / highlight.js を使用（frontend 判断）
- 言語指定なしのときは `text`（ハイライトなし）

#### 区切り線（カスタムブロック）

- `<hr>` 相当。`border-t border-border` の水平線
- クリック・フォーカス: ブロック選択状態（薄いハイライト）
- キャプションなし

#### 画像ブロック

詳細は §8（画像 UX）を参照。

## 7. インライン装飾とフローティングツールバー

### 7-1. 対応インライン装飾

| 装飾 | BlockNote 標準 | ショートカット |
|------|--------------|--------------|
| **太字** | ✓ | `Ctrl/Cmd + B` |
| *斜体* | ✓ | `Ctrl/Cmd + I` |

| ~~取消線~~ | ✓ | — |
| `インラインコード` | ✓ | `Ctrl/Cmd + E` |
| リンク | ✓ | `Ctrl/Cmd + K` |

### 7-2. フローティングツールバー

テキストを選択するとカーソル上部に表示される（BlockNote のFormattingToolbar 相当）。

#### 構成・順序（左から右）

```
┌─────────────────────────────────────────────────────────────────────┐
│  B  │  I  │  U  │  S  │  </>  │  ─  │  🔗  │ ▼ブロック変換 │
└─────────────────────────────────────────────────────────────────────┘
    太字  斜体  下線  取消線  コード  区切線  リンク   変換
```

| 位置 | アイコン / ラベル | 機能 | aria-label |
|------|----------------|------|-----------|
| 1 | **B** | 太字 | 太字 |
| 2 | *I* | 斜体 | 斜体 |
| 3 | U（下線あり） | 下線 | 下線 |
| 4 | ~~S~~ | 取消線 | 取消線 |
| 5 | `</>` | インラインコード | インラインコード |
| 6 | `🔗` | リンク挿入/編集（URL入力ポップオーバー） | リンクを挿入 |
| 7 | ブロック変換ボタン | 現在のブロック種別をドロップダウンで変換 | ブロックタイプを変更 |

- ツールバー背景: `bg-popover`、`border border-border`、`rounded-md shadow-md`
- ボタン間: `gap-0.5`、各ボタン `h-7 w-7` 程度（`icon-sm` サイズ相当）
- アクティブ（選択中の装飾がオン）: `bg-muted text-foreground`

### 7-3. リンク挿入ポップオーバー

「リンクを挿入」ボタンを押すと URL 入力欄を持つ小さなポップオーバーが表示される。

```
┌─────────────────────────────────┐
│  URL を入力           [適用]    │
└─────────────────────────────────┘
```

- `input[type="url"]` で URL を入力
- `placeholder="https://..."` 
- Enter キーまたは「適用」ボタンで確定
- Escape で閉じる
- 既存リンクをクリック: 同ポップオーバーが開き編集・削除ができる

## 8. `/` スラッシュコマンドメニュー

### 8-1. 表示トリガー

- 行の先頭で `/` を入力した瞬間（BlockNote 標準の `SuggestionMenuController` を使用）
- 入力中のテキストでフィルタリング（例: `/見出` → 見出しブロックが候補に）
- `Escape` で閉じ、`/` テキストを残す（キャンセル）

### 8-2. メニュー構成

グループ分けして表示する。BlockNote の `getDefaultSlashMenuItems` をベースに、
カスタムブロック（引用・区切り線）を追加する。

#### グループ: 基本ブロック

| アイコン | 項目名 | キーワード | 説明 |
|---------|--------|-----------|------|
| `¶` | 段落 | paragraph, text | 通常テキスト |
| `H1` | 見出し 1 | heading, h1 | 大見出し |
| `H2` | 見出し 2 | heading, h2 | 中見出し |
| `H3` | 見出し 3 | heading, h3 | 小見出し |

#### グループ: リスト

| アイコン | 項目名 | キーワード |
|---------|--------|-----------|
| `•` | 箇条書き | bullet, list, ul |
| `1.` | 番号付きリスト | numbered, list, ol |
| `☐` | チェックリスト | check, todo |

#### グループ: その他

| アイコン | 項目名 | キーワード |
|---------|--------|-----------|
| `"` | 引用 | quote, blockquote |（カスタムブロック）
| `{ }` | コードブロック | code, pre |
| `—` | 区切り線 | divider, hr, rule |（カスタムブロック）
| `🖼` | 画像 | image, photo |

### 8-3. メニュー外観

```
┌───────────────────────────────────┐
│  /                                 │
├───────────────────────────────────┤
│  基本ブロック                       │
│  ┌──┐  段落                        │
│  │¶ │  通常テキストを入力           │
│  └──┘                              │
│  ┌──┐  見出し 1                    │
│  │H1│  大きな見出し                 │
│  └──┘                              │
│  ...                               │
│  リスト                             │
│  ...                               │
└───────────────────────────────────┘
```

- 背景: `bg-popover`、`border border-border`、`rounded-md shadow-lg`
- 幅: `w-72`（288px）
- 最大高さ: `max-h-80 overflow-y-auto`
- グループラベル: `text-xs text-muted-foreground font-medium px-2 py-1`
- 各アイテム: `flex gap-3 px-2 py-2 rounded-sm hover:bg-muted cursor-pointer`
  - 左: アイコン領域 `w-8 h-8 bg-muted rounded-sm`（中央にアイコン文字）
  - 右: タイトル（`text-sm font-medium`）+ 説明（`text-xs text-muted-foreground`）
- 選択中（キーボードフォーカス）: `bg-muted`

### 8-4. キーボード操作

- `↑` / `↓`: 項目移動
- `Enter`: 項目を選択してブロック挿入
- `Escape`: メニューを閉じる（`/` を残す）
- 文字入力: リアルタイムフィルタリング
- フィルタ結果が0件: 「一致するコマンドはありません」を表示（ただし `/` は通常テキストとして扱われる）

## 9. 画像 UX

### 9-1. 挿入方法

#### 方法 A: `/` コマンドメニューから

1. 行頭で `/` → 「画像」を選択 → 画像ブロックが挿入され、ファイル選択ダイアログが開く

#### 方法 B: ドラッグ＆ドロップ

1. エディタ上に画像ファイルをドロップ
2. カーソル位置（ドロップ位置に最も近いブロック境界）に画像ブロックが挿入される
3. 即アップロード処理を開始

#### 方法 C: クリップボードからの貼り付け

1. 画像をコピーして `Ctrl/Cmd + V`
2. カーソル位置に画像ブロックが挿入され即アップロード

### 9-2. アップロードフロー（UX 視点）

```
ファイル選択 / ドロップ / 貼り付け
    │
    ▼ ブラウザ側チェック（即時・エラーは §9-4）
    │ ・形式（JPEG/PNG/WebP）
    │ ・サイズ（原画 20MB 以下）
    │
    ▼ [アップロード中プレースホルダ表示]  ← §9-3
    │ ・ブラウザ側圧縮（lib/images/compress.ts）
    │ ・Server Action: createUploadUrl()（署名付き URL 取得）
    │ ・PUT で Supabase Storage に直接アップロード
    │
    ▼ 成功: 画像ブロックに URL をセット、プレースホルダを画像に置換
    │ 失敗: [失敗プレースホルダ表示]  ← §9-4
```

### 9-3. アップロード中プレースホルダ

```
┌────────────────────────────────────────────────────┐
│                                                    │
│          [Loader2 回転アイコン]                     │
│          画像をアップロード中…                      │
│                                                    │
│          ████████░░░░░░░░  60%                     │
└────────────────────────────────────────────────────┘
```

- 高さ: `h-32`（固定。アスペクト比確定後に本来の高さに移行する必要はない）
- 背景: `bg-muted rounded-md`
- 進捗バー: XHR/fetch の `onProgress` イベントで更新（パーセンテージ表示は任意。実装コストが高ければローダーアイコンのみで可）
- テキスト: 「画像をアップロード中…」(`text-sm text-muted-foreground`)
- キャンセルボタン: **第1弾ではなし**（実装コスト対効果が低い）

### 9-4. エラー表示と文言

#### 形式エラー（サポートされていない形式）

```
┌────────────────────────────────────────────────────┐
│  ⚠  この形式の画像はサポートされていません          │
│     JPEG / PNG / WebP 形式のファイルを選択してく    │
│     ださい。（GIF は非対応）                        │
│                                                    │
│  [画像を選択し直す]                                │
└────────────────────────────────────────────────────┘
```

定数名案: `ERR_IMAGE_UNSUPPORTED_FORMAT`（`lib/constants/errors.ts` に追加）

#### サイズエラー（原画 20MB 超過）

```
┌────────────────────────────────────────────────────┐
│  ⚠  ファイルサイズが大きすぎます                    │
│     20MB 以下の画像を選択してください。             │
│                                                    │
│  [画像を選択し直す]                                │
└────────────────────────────────────────────────────┘
```

定数名案: `ERR_IMAGE_TOO_LARGE`（`lib/constants/errors.ts` に追加）
※ 数値は `MAX_IMAGE_INPUT_SIZE_MB` 定数から生成するテンプレート関数とすること:
`ERR_IMAGE_TOO_LARGE = (max: number) => \`${max}MB 以下の画像を選択してください\``

#### 容量上限エラー（無料 200MB / 有料 5GB 到達）

無料プランの場合:
```
┌────────────────────────────────────────────────────┐
│  ⚠  ストレージの上限（200MB）に達しました           │
│     不要な画像を含むページを削除するか、プレミア     │
│     ムプランへのアップグレードをご検討ください。     │
│                                                    │
│  [アップグレードを検討する]   [キャンセル]           │
└────────────────────────────────────────────────────┘
```

有料プランの場合:
```
┌────────────────────────────────────────────────────┐
│  ⚠  ストレージの上限（5GB）に達しました             │
│     不要な画像を含むページを削除してください。       │
│                                                    │
│  [キャンセル]                                      │
└────────────────────────────────────────────────────┘
```

定数名案:
- `ERR_IMAGE_STORAGE_LIMIT_FREE = (max: number) => \`ストレージの上限（${max}MB）に達しました\``
- `ERR_IMAGE_STORAGE_LIMIT_PAID = (max: number) => \`ストレージの上限（${max}GB）に達しました\``
（`lib/constants/errors.ts` に追加）

「アップグレードを検討する」は第2弾の課金ページへのリンク。第1弾では `/settings/usage` または `/settings` へリンクしてよい。

#### アップロード失敗（ネットワークエラー等）

ブロック内にインラインで表示:

```
┌────────────────────────────────────────────────────┐
│  ⚠  アップロードに失敗しました                      │
│     ネットワーク状態を確認して再試行してください。   │
│                                                    │
│  [再試行]  [削除]                                   │
└────────────────────────────────────────────────────┘
```

定数名案: `ERR_IMAGE_UPLOAD_FAILED`（`lib/constants/errors.ts` に追加）

- 「再試行」: 同じファイルで最初からアップロードを再実行
- 「削除」: このブロックを削除してエディタに戻る

### 9-5. 圧縮後表示

アップロード成功後の画像ブロック:

```
┌────────────────────────────────────────────────────┐
│                                                    │
│              [画像表示]                             │
│         （max-w-full, rounded-md）                  │
│                                                    │
│  キャプションを入力（任意）                          │
└────────────────────────────────────────────────────┘
```

- 画像は `max-width: 100%`（エディタ幅に収まる）・`height: auto`
- 丸み: `rounded-md`
- キャプション: 画像ブロック下部の `text-sm text-muted-foreground` のインライン編集テキスト
  - BlockNote の `ImageBlock` がキャプションをサポートしているか要確認。なければカスタムで追加
  - キャプションは第1弾では **任意**（空でも構わない）
- 画像クリック: 特別な動作なし（第1弾。モーダル拡大は第2弾以降で検討）

## 11. エッジケース

### 11-1. 空状態

- 新規作成直後: 本文は空（`content: []`）。§10-1 のプレースホルダを表示
- タイトルのみ入力: 本文プレースホルダのまま表示

### 11-2. ローディング

- BlockNote の `next/dynamic`（`ssr:false`）読み込み中: `EditorSkeleton` を表示
  - スケルトン: `h-4 bg-muted rounded animate-pulse` の行が 5〜6 本、各行の幅をランダムに変える
  - 上部にページヘッダのスケルトン（アイコン+タイトル部分）は M2 実装済み外なので不要

### 11-3. 保存エラー

- §5（自動保存ステータス）の `error` 状態に移行
- エディタは通常通り使い続けられる（ページ全体を落とさない）
- 次回の debounce 発火時に再保存を試みる

### 11-4. 本文サイズ上限（1MB）

Server Action `updatePageContent` は 1MB チェックを行う（M2 実装済み）。
エラーが返った場合:
- 自動保存ステータスが `error` に遷移
- ステータス表示の文言: 「ページが大きすぎて保存できません（上限 1MB）。不要なコンテンツを削除してください。」
- 定数名案: `ERR_PAGE_CONTENT_TOO_LARGE`（既存定数を流用）

### 11-5. 権限なし / 存在しないページ

`app/(main)/pages/[id]/page.tsx` で `notFound()` を呼ぶ設計は M2 実装済み。
エディタ層では対処不要。

### 11-6. オフライン

ブラウザがオフラインの場合、Server Action が失敗 → `error` 状態に遷移。
「再試行」は手動。オフライン専用メッセージは第1弾では出さない（エラー文言で代用）。

### 11-7. 複数タブで同一ページを開いた場合

要件定義書 §3.3.2 に「last-write-wins、競合解決は第3弾」と明記。
UI 上の特別な表示はしない。

### 11-8. 画像アップロード中にページ遷移

アップロード中（非同期処理中）に別ページへ遷移した場合、アップロードは中断される。
第1弾では「離脱確認ダイアログ」は出さない（実装コスト対効果が低い）。
アップロード中のプレースホルダブロックは未保存のまま消える（本文自体も未保存の場合）。

## 12. キーボード操作とアクセシビリティ

### 12-1. エディタ内キーボード操作（BlockNote 標準）

| 操作 | キー |
|------|------|
| ブロック種別変換（インライン） | `#` + スペース → H1、`##` + スペース → H2 等（Markdown ショートカット） |
| リスト開始 | `-` + スペース → 箇条書き、`1.` + スペース → 番号付き |
| チェックリスト | `[]` + スペース |
| 引用 | `>` + スペース（カスタムブロックで実装） |
| コードブロック | ` ``` ` + Enter（要確認） |
| Tab / Shift+Tab | リストのインデント変更 |
| Enter | 次のブロックを同種で作成（リストは同種継続、見出しは段落に戻る） |
| Backspace（行頭） | ブロック種別を段落に戻す or 前のブロックと結合 |
| Ctrl/Cmd + Z | アンドゥ |
| Ctrl/Cmd + Y または Ctrl/Cmd + Shift + Z | リドゥ |

### 12-2. グローバルキーボードショートカット（エディタページ全体）

| 操作 | キー | 備考 |
|------|------|------|
| 新規ページ作成 | — | 第1弾では未実装（サイドバーの「＋」を使う） |
| フォーカスをタイトルへ | `Ctrl/Cmd + Shift + T`（案） | PageHeader のタイトル input にフォーカスを移す |
| フォーカスをエディタへ | `Ctrl/Cmd + Shift + E`（案） | エディタの先頭ブロックにフォーカス |

**frontend に要確認**: BlockNote がデフォルトで `Ctrl/Cmd + Shift + T` 等を使っているか確認してショートカットが衝突しないようにすること。

### 12-3. フォーカス管理

- ページ読み込み完了後: タイトル input に自動フォーカス（新規作成後はタイトルを先に入力するフローが自然）
- タイトルで Enter: フォーカスがエディタの先頭ブロックへ移動（BlockNote の操作性と合わせる）
  - これは PageHeader の `handleTitleKeyDown` に `onEnterPress` コールバックを追加する形で実現する。frontend で実装すること

### 12-4. ARIA・スクリーンリーダー対応

- エディタ全体: `role="textbox"` + `aria-multiline="true"` + `aria-label="ページ本文"`
  - ただし BlockNote が内部で適切な role を付与している場合はそちらに委ねる
- 自動保存ステータス: `role="status"` + `aria-live="polite"`（エラー時は `role="alert"` + `aria-live="assertive"`）
- フローティングツールバー: `role="toolbar"` + `aria-label="テキスト書式設定"`
- スラッシュメニュー: `role="listbox"` + `aria-label="コマンドを選択"` + `aria-activedescendant`

### 12-5. コントラスト比

- 本文テキスト: `text-foreground`（ライト: `#0a0a0a` 相当）→ 背景白に対して十分なコントラスト
- プレースホルダ: `text-muted-foreground/50`（薄い）→ WCAG AA 未達になる可能性あり。実装者は実測してコントラスト比 3:1 以上（大テキストは 3:1、小テキストは 4.5:1）を確認すること
- エラー文言: `text-destructive` → 背景に対して 4.5:1 以上

### 12-6. フォーカスリング

既存の shadcn/ui スタイル（`focus-visible:ring-3 focus-visible:ring-ring/50`）を踏襲。
エディタ内部は BlockNote のデフォルトフォーカス表示を使用。

### 12-7. data-testid 候補（E2E テスト用）

| 要素 | `data-testid` |
|------|--------------|
| エディタルート要素 | `editor-root` |
| 自動保存ステータス | `autosave-status` |
| 再試行ボタン | `autosave-retry` |
| 画像アップロードプレースホルダ | `image-upload-placeholder` |
| 画像アップロードエラー | `image-upload-error` |
| 画像再試行ボタン | `image-retry` |
| 画像削除ボタン（エラー時） | `image-delete` |
| スラッシュメニュー | `slash-command-menu` |
| フローティングツールバー | `formatting-toolbar` |

## 15. 申し送り事項

### backend エンジニアへ

- `lib/constants/errors.ts` に以下の定数を追加してほしい（§9-4 参照）:
  - `ERR_IMAGE_UNSUPPORTED_FORMAT`
  - `ERR_IMAGE_TOO_LARGE = (max: number) => ...`
  - `ERR_IMAGE_STORAGE_LIMIT_FREE = (max: number) => ...`
  - `ERR_IMAGE_STORAGE_LIMIT_PAID = (max: number) => ...`
  - `ERR_IMAGE_UPLOAD_FAILED`
- `lib/constants/limits/` に `AUTOSAVE_DEBOUNCE_MS = 1500` を追加してほしい（`editor.ts` 新規または `pages.ts` に追記）

### frontend エンジニアへ

- BlockNote の引用・区切り線のカスタムブロック実装（`createReactBlockSpec`）が必要
- `AppShell` の `header` に自動保存ステータスを表示するための接続方法を決めること（Props スロット vs Context）
- `PageHeader` にエディタへのフォーカス移動コールバック（タイトルで Enter を押した時）を追加すること
- BlockNote の FormattingToolbar・SuggestionMenu は標準 API をカスタマイズして使う
- `EditorSkeleton` の高さ・行数は実装者が UX 的に自然な値を選んでよい
- 画像ブロックのキャプション機能が BlockNote 標準でサポートされているか確認し、なければカスタム実装すること
- モバイルでフローティングツールバーが画面外にはみ出す場合の位置調整を実装すること
- `data-testid` 一覧（§12-7）をコンポーネントに付与すること

### tester エンジニアへ

- §12-7 の `data-testid` を前提に E2E テストを記述すること
- 画像エラーの E2E は Supabase Storage をモック（`page.route()`）すること
- 自動保存ステータスの遷移テストは `vi.useFakeTimers()` で `saved` → `idle` の 3 秒タイマーも検証すること
