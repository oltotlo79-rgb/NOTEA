# M7: 共有機能（第2弾・2番目の機能）実装計画

> 要件定義書 3.8、.claude/rules/ 準拠。検索（M6）に続く第2弾機能。

**Goal:** ページ単位で共有リンクを発行し、リンクを知っている人が `/share/[token]` で閲覧（匿名OK・読取専用）できる。「編集可」リンクは Notea ログインユーザーのみ編集可能。発行・失効（revoke）に対応。

## 確定した設計判断（ユーザー合意）
- **閲覧リンク**: 匿名OK・読取専用。
- **編集リンク**: ログイン必須（`auth.uid()` で書き込みを認証）。匿名編集は不可。
- **リンク管理**: 発行・失効（revoke）のみ。有効期限は設けない。
- 共有単位: 単一ページのみ（サブページは含めない）。要件「ページ単位」。

## セキュリティ設計（肝）
匿名/非所有者は pages の owner RLS を通れないため、**トークン経由のアクセスは SECURITY DEFINER 関数に集約**する。
RLS はトークンを知り得ない（auth コンテキストに無い）ため、トークン検証は関数内で行い RLS には載せない。

- `page_shares` テーブル: 所有者のみ select/insert/delete（自分の共有を管理）。RLS owner ポリシー。
- `get_shared_page(p_token)` SECURITY DEFINER: token → ページ（id, title, icon, content, content_text, updated_at, permission）を返す。view/edit 両対応。`search_path=''` 固定。anon/authenticated に execute 付与。
- `update_shared_page(p_token, p_content, p_content_text)` SECURITY DEFINER: 編集リンク（permission='edit'）かつ **呼び出し元が認証済み（auth.uid() is not null）** の場合のみ pages を更新。それ以外は例外。authenticated のみ execute（anon には付与しない）。
- 画像: Storage は owner プレフィックス RLS のため匿名は署名URLを作れない。`getSharedImageUrl(token, path)` Server Action が token を検証し、`lib/supabase/admin.ts`（service_role）で署名URLを発行する（「セッションの無い共有閲覧」は admin 利用の正当ケース。理由をコメント明記）。path が共有ページ所有者のプレフィックス配下であることを必ず検証（他人の画像を引けないように）。

## 担当・ラウンド
- Round 1: backend（migration + RPC + 共有管理/アクセス actions + 定数 + 単体）
- Round 2: frontend（/share/[token] 公開ルート・読取ビュー・共有ダイアログ UI・編集導線）
- Round 3: tester（単体補強 + E2E）
- Round 4: evaluator（ゲート・RLS/トークン漏洩・BYOK 不変）

## タスク

### M7-T1（backend）
- `supabase/migrations/<ts>_sharing.sql`:
  - `page_shares`（id, page_id fk cascade, user_id fk cascade, token text unique, permission check in ('view','edit'), created_at）。`unique(page_id, permission)` で1ページ＝view/edit 各1本。
  - RLS 有効化 + owner ポリシー（select/insert/delete。update は不要＝失効は delete）。
  - `get_shared_page` / `update_shared_page` SECURITY DEFINER（search_path=''）。
  - GRANT: `grant select,insert,delete on page_shares to authenticated`（+ service_role）。`grant execute on get_shared_page to anon, authenticated`。`grant execute on update_shared_page to authenticated`。
  - 型再生成（Git Bash + ダミー SUPABASE_ACCESS_TOKEN）。
- `lib/actions/shares.ts`（owner 管理, ActionResult）: `createShare(pageId, permission)` / `revokeShare(pageId, permission)` / `listShares(pageId)`。認証→Zod→所有ページ確認→token 生成（crypto 乱数）→upsert/delete。
- `lib/actions/shared-pages.ts`（共有アクセス）:
  - `getSharedPage(token)`（custom shape: `{ data: SharedPage | null; error? }`、@module に理由）— anon クライアントで `.rpc('get_shared_page')`。
  - `updateSharedPageContent(token, content, contentText)`（ActionResult）— requireUser（ログイン必須）→ Zod/サイズ →`.rpc('update_shared_page')`。
  - `getSharedImageUrl(token, path)`（custom shape）— token 検証 + admin 署名URL。
- 定数: `lib/constants/limits/share.ts`（SHARE_TOKEN_BYTES など）、`errors.ts`（ERR_SHARE_*）、`routes.ts`（SHARE プレフィックス。**PROTECTED_PATHS には入れない**＝公開ルート）。
- 単体: createShare/revokeShare/listShares（未認証/他人ページ/正常）、getSharedPage（無効token→null）、updateSharedPageContent（未ログイン拒否/view リンク拒否/正常）、トークン・鍵がレスポンスに混ざらないこと。

### M7-T2（frontend）
- `app/(public)/share/[token]/page.tsx`（RSC・noindex）: getSharedPage → 無ければ notFound。view→読取ビュー。edit→ログイン状態を確認しログイン済みなら編集ビュー、未ログインなら読取＋「編集するにはログイン」バナー。
- `components/share/SharedPageView.tsx`（client）: BlockNote を `editable={false}` で読取表示（既存 Editor を読取モード対応 or 専用 viewer）。画像は resolveFileUrl を getSharedImageUrl 経由に差し替え。
- `components/share/SharedPageEditor.tsx`（client）: 編集リンク用。autosave を `updateSharedPageContent(token, …)` に向ける。
- `components/share/ShareDialog.tsx`（client）: PageHeader の「共有」ボタンから開く。view/edit リンクの発行・URL コピー・失効。listShares/createShare/revokeShare を呼ぶ。
- PageHeader に共有ボタン追加。`lib/constants/routes.ts` の SHARE を使い URL 生成（NEXT_PUBLIC_APP_URL）。

### M7-T3（tester）
- 単体: ShareDialog（発行→URL 表示→コピー→失効）、SharedPageView（読取描画・編集不可）。
- E2E `share.spec.ts`: ページ作成→共有ダイアログで view リンク発行→別コンテキスト（未ログイン）で開く→本文が見える・編集不可。edit リンク→未ログインは編集不可、ログイン済みは編集できる。失効後は 404。
- カバレッジ閾値維持・E2E 2回連続緑。

### M7-T4（evaluator）
全ゲート＋「匿名が他人の非共有ページを読めない」「view リンクで書き込めない」「token/鍵がレスポンス・ログに出ない」「PROTECTED_PATHS 不変で /share が公開」を判定。

## 完了条件
- view リンクで匿名が読取専用閲覧でき、edit リンクでログインユーザーが編集できる
- 失効後リンクは 404、非共有ページは匿名から不可視
- 全テスト緑・カバレッジ閾値・既存機能と BYOK 不変
