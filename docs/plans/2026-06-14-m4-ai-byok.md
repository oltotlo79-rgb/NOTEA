# M4: AI（BYOK）実装計画

> チーム分業（PM 統括）。要件定義書 3.4 / 5.3 / 5.6、**`.claude/rules/ai-byok.md`（セキュリティ正本）** 厳守。

**Goal:** ユーザー自身の API キー（ブラウザ localStorage のみ保存）で AI 機能（要約・続き書き・翻訳・ページへの質問）を提供する。サーバーは「使った回数」だけを記録し、鍵・プロンプト・応答は一切持たない。

**最重要原則（ai-byok.md）:** 鍵を DB/ログ/Sentry/cookie/URL/Server Action 引数に**載せない**。consumeAiUsage の引数は provider のみ。OpenAI パススルーは Authorization で受けて転送後破棄・no-store・allowlist のみ。

**前提（M1 完了済み）:**
- ai_usage テーブル（RLS・鍵/プロンプト/応答カラムなし）+ rpc consume_ai_usage(p_provider, p_limit)（アトミック・上限超過で例外）+ authenticated/service_role GRANT
- lib/constants/limits/ai.ts: AI_PROVIDERS・FREE_AI_PROVIDERS=['gemini']・FREE_AI_DAILY_LIMIT=5・PAID_AI_DAILY_LIMIT=100・型 AiProvider
- lib/constants/security.ts: AI_PROVIDER_API_ORIGINS（CSP connect-src＋proxy allowlist 共用）。proxy CSP は AI 3ドメイン許可済み

---

## 消費（回数カウント）の設計 — 二重カウント防止

consume_ai_usage rpc を呼ぶ consumeAiUsage(userId, provider) を lib/services/usage.ts に追加。消費点は provider ごとに必ず1回:
- Gemini / Anthropic（ブラウザ直呼び）: クライアントが Server Action consumeAiUsage(provider) を呼ぶ → 成功 → lib/ai/providers/{provider}.generate() でプロバイダ API を直接叩く。
- OpenAI（CORS非対応 → パススルー）: クライアントは Server Action を呼ばず、/api/ai/proxy に鍵を Authorization で渡す → proxy が内部で consumeAiUsage(userId,'openai') を消費してから OpenAI へ転送。

各 provider の消費は1回だけ。プラン検証（無料は gemini のみ・上限）は Server Action と proxy の両方でサーバー側実施（UI 出し分けに頼らない）。

### M4-T2（backend）: 消費・パススルー
- lib/services/usage.ts: consumeAiUsage(userId, provider) — プラン取得（無料は FREE_AI_PROVIDERS 検証＝gemini 以外拒否）→ 上限を引数に rpc consume_ai_usage をアトミック呼び出し。上限超過/プラン違反は判別可能なエラー。getAiUsageToday(userId) — 本日の provider 別 count と残回数。
- lib/actions/ai.ts（'use server'、@module）: consumeAiUsage(provider: AiProvider): ActionResult<{ remaining: number }> — 認証→Zod（provider のみ）→services.consumeAiUsage。引数に apiKey 等を絶対に持たせない。getAiUsageToday()。
- app/api/ai/proxy/route.ts: POST。supabase.auth.getUser() 必須→services.consumeAiUsage(userId,'openai')→転送先は AI_PROVIDER_API_ORIGINS の OpenAI ドメインのみ（allowlist・SSRF 防止）→Authorization の鍵をそのまま転送して変数に保持せず破棄、Cache-Control: no-store、リクエスト/レスポンスボディ・鍵をログ/例外に一切残さない。
- テスト: consumeAiUsage（未認証/無料で openai 指定→拒否/上限ちょうど/上限超過/日付）、proxy（未認証401/allowlist外403/鍵がレスポンス・ログに含まれない）。
