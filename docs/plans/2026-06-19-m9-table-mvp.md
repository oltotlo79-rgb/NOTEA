# M9: 表（データベース）機能 MVP 実装計画＋設計

> 要件定義書 3.7。第2弾の最後・最重量機能。今回は **MVP（テーブル表示のみ・型付き列）** を実装し、ボード/カレンダー表示は後続に切る。

## スコープ
- ページ内に「データベース表ブロック」を挿入できる。
- 列に型を持たせる: テキスト / 数値 / チェックボックス / 選択肢 / 日付（要件の初期セット）。
- テーブル表示で行の追加・編集・削除、列の追加・リネーム・型変更・削除ができる。
- **ボード（カンバン）/ カレンダー表示も実装済み**（2026-06-19 追加）:
  - ボード: select 列でレーン分け・カードのレーン変更・レーンへのカード追加。
  - カレンダー: date 列で月グリッド配置・前後月ナビ・日への行追加。
  - ドラッグ&ドロップは未対応（レーン変更は select、日付移動はセル編集で行う）。

## 設計

### データの持ち方
BlockNote のカスタムブロック（`createReactBlockSpec`）として実装する。block の props は
プリミティブのみ対応のため、表データ全体を **1 つの string prop `data` に JSON 文字列**で保存する。
これにより新規 DB テーブル・マイグレーション不要で、既存の `content`(jsonb) 保存・autosave に乗る。

```
type DataTable = {
  columns: { id: string; name: string; type: ColumnType; options?: string[] }[]
  rows: { id: string; cells: Record<columnId, CellValue> }[]
}
type ColumnType = 'text' | 'number' | 'checkbox' | 'select' | 'date'
// CellValue: text/select/date=string, number=number|null, checkbox=boolean
```

ブロック名は BlockNote 標準の `table` と衝突しないよう **`dataTable`** とする。

### レイヤ
- `lib/editor/data-table.ts`: **純粋関数**（addColumn/removeColumn/renameColumn/changeColumnType(値の型変換含む)/addRow/removeRow/setCell/parse/serialize/extractPlainText）。ブラウザ/サーバー中立・単体テストの主対象。
- `lib/editor/schema.ts`: `BlockNoteSchema.create({ blockSpecs: { ...defaultBlockSpecs, dataTable } })`。Editor が使う拡張スキーマ。
- `components/editor/blocks/DataTableBlock.tsx`: `createReactBlockSpec` の render。型別セル入力（テキスト/数値 input・checkbox・select・date）。列/行操作 UI。`editor.updateBlock` で `data` を更新→autosave。
- `components/editor/Editor.tsx`: 拡張スキーマを使用。スラッシュメニューに「データベース」項目追加。`extractText` を拡張し dataTable の列名・セル文字列も `content_text` に含める（M6 検索でヒットさせる）。

### 検索との整合
`extractText`（content_text 抽出）は標準ブロックを `blocksToMarkdownLossy` で処理するが、
カスタムブロックは拾えないため、`dataTable` ブロックは `extractPlainText(data)` で列名＋セル値を
連結して追記する。これで表の中身も全文検索の対象になる。
