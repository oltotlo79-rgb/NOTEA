# M6: 検索（第2弾・最初の機能）実装計画

> チーム分業（PM 統括）。要件定義書 3.9 / 6.3、.claude/rules/ 準拠。第2弾の口火として外部依存のない検索から着手。

**Goal:** 自分のページをタイトル・本文（content_text）で全文検索し、`/search` で結果を表示する。カーソルページネーション・RLS で自分のページのみ・ごみ箱除外。

**前提（第1弾完了済み）:** pages.content_text は保存時に同期済み（updatePageContent）。RLS owner ポリシー。カーソルページネーション基盤。

**方針:** PostgreSQL `pg_trgm` 拡張＋GIN インデックスで日本語含む部分一致検索（pg_bigm はローカル Supabase に無い可能性があるため pg_trgm を基本とし、backend が利用可能な最適手段を選ぶ）。第2弾で初の新規マイグレーション。

---

## 担当・ラウンド
- Round 1: backend（検索マイグレーション＋search action/service）
- Round 2: frontend（/search ページ・検索ボックス UI・ショートカット）
- Round 3: tester（search 単体＋E2E）
- Round 4: evaluator（ゲート）

## タスク

### M6-T1（backend）
- supabase/migrations/<ts>_search.sql: `create extension if not exists pg_trgm`。pages に GIN インデックス（title gin_trgm_ops, content_text gin_trgm_ops）。必要なら `search_text` 生成列（title||' '||content_text）＋GIN。authenticated は既存 GRANT で select 可。RLS は既存 owner_select が効く。
- types 再生成（スキーマ変化があれば。Git Bash + ダミー SUPABASE_ACCESS_TOKEN 回避策）。
- lib/actions/search.ts（'use server'、@module に custom shape 理由）: `searchPages(query: string, cursor?: string): { data: SearchResult[]; nextCursor?: string; error? }`。認証→Zod（query 1..100、空はスキップ）→ pages を user_id=自分・is_trashed=false で title/content_text 部分一致（ILIKE '%q%' or trgm similarity）検索、updated_at desc + id desc のカーソル、limit 定数（SEARCH_PAGE_SIZE）。結果は id/title/icon/updated_at＋スニペット（content_text から一致箇所周辺を抜粋、無ければ先頭）。`.eq('user_id', userId)` 多層防御。
- 定数: lib/constants/limits に SEARCH_PAGE_SIZE（例 20）・MAX_SEARCH_QUERY_LENGTH（100）。errors.ts に必要分。
- テスト: searchPages（未認証/空クエリ/正常一致/カーソル/他人ページ非表示=RLS）。test-utils 活用。

### M6-T2（frontend）
- app/(main)/search/page.tsx（RSC 枠）＋ components/search/SearchView.tsx（client）: 検索ボックス（debounce）→ searchPages → 結果リスト（アイコン＋displayTitle＋スニペット、クリックで /pages/[id]）。空/ローディング/0件状態。robots noindex（アプリ内）。
- 検索導線: サイドバーまたはトップに検索ボタン、グローバルショートカット（例 Ctrl/Cmd+K）で /search へ。use-global-shortcuts に追加。
- routes.ts に SEARCH 追加、PROTECTED_PATHS に /search 追加。
- スニペットの一致語ハイライト（任意・XSS 注意＝dangerouslySetInnerHTML 不使用、テキスト分割で）。

### M6-T3（tester）
- 単体: SearchView（入力→action 呼び出し・結果描画・空状態）、search action 補強。
- E2E search.spec.ts: ページ作成（一意タイトル）→ /search で検索→ヒット→クリックで遷移。他ユーザーのページが出ないこと（seed のみなので自分の範囲）。0件表示。
- カバレッジ閾値維持・E2E 2回連続全緑。

### M6-T4（evaluator）
全ゲート＋RLS（自分のページのみ）・カーソルページネーション・noindex・規約・BYOK 不変を判定。

## 完了条件
- /search で自分のページをタイトル・本文から検索でき、結果から遷移できる
- 他人のページは出ない（RLS）・ごみ箱は除外・カーソルで追加読み込み
- 全テスト緑・カバレッジ閾値・既存機能と BYOK 不変
